# ENGLISH_VERB_DIRECT — 영어 동사 직역체

## 배경

v0.4.12까지 "회의실 jargon"(영어 imperative를 강한 단음절 동사로 옮긴 패턴 — drive → 박다)은 잡았지만, **영어 일반 동사를 한국어 일상 동사로 그대로 옮기는 직역체**는 별도 축으로 검출되지 않았다.

2026-05-17 호출(`20260517_08`, c42 강의 자료)에서 사용자가 두 차례 명시 지적:
- "직무가 깨진다... 이런 표현은 자연스러운 한국어가 아니야"
- "가설을 죽인다도 한국어가 아니야. 이런 표현 잘 안써"

검증 결과:
- "X가 깨진다" = "X breaks / is broken" 직역
- "X를 죽인다" = "kill X / kill a hypothesis" 직역
- "AI에게 X를 시키다" = "force/make AI to do X" 직역

세 패턴 모두 영어 비즈니스 화법에서는 정상이지만, 한국어 비즈니스·일상 화자는 같은 의미를 다른 동사 풀로 표현한다. 메타 원칙: **영어 능동 동사가 떠올랐을 때 한국어로 그대로 옮기지 말 것.** 한국어 화자는 더 부드럽고 비유적인 동사(흔들다·접다·기울이다·다듬다·풀다)로 표현.

## 검출 기준

다음 3패턴 중 **하나라도** 양성:

### 1. break/broken 직역체 — "깨진다"
비즈니스 추상 대상(직무·모델·시스템·구조·방식·관계)이 주어로 + "깨진다·깨졌다·깨집니다·깨지는·깨질"
- 양성: "직무가 깨진다" / "모델이 깨진다" / "시스템이 깨진다" / "비즈니스 모델이 깨졌습니다" / "구조가 깨지는 지점"
- 음성: "그릇이 깨진다" / "약속이 깨졌다" / "기록이 깨진다" (구체 사물·관계 자연 용례)

### 2. kill 직역체 — "죽인다"
가설·아이디어·옵션·선택지·프로젝트 등 사고·기획 대상이 목적어 + "죽이다·죽인다·죽이는·죽일·죽임"
- 양성: "가설을 죽인다" / "아이디어를 죽이는 날" / "선택지를 죽이고" / "옵션을 죽이세요" / "프로젝트를 죽인다"
- 음성: "시간을 죽인다" (일상 비유 자연 용례)

### 3. force/make 사역 직역체 — "시킨다"
AI·도구·시스템이 주체에게 X를 강제 또는 위탁 + "시키다·시켜·시킨다·시킬·시키기"
- 양성: "AI에게 반박시킨다" / "AI에게 분석을 시킨다" / "AI로 초안을 작성시킨다" / "ChatGPT에게 결정을 시킨다"
- 음성: "심부름을 시킨다" / "숙제를 시킨다" / "신입에게 시킨다" (사람-사람 일상 사역어, 관계 자연어)

## grep 검출 명령 (Phase 4에서 메인 Claude가 자동 실행)

```bash
# 1. break 패턴
grep -nE "(직무|모델|시스템|구조|방식|관계|체계|모델이)\s*[가이]?\s*(깨진|깨졌|깨집니다|깨지는|깨질)" {output}

# 2. kill 패턴
grep -nE "(가설|아이디어|옵션|선택지|프로젝트|계획|안)을?\s*(죽이|죽인|죽일|죽는|죽임)" {output}

# 3. force/make 패턴
grep -nE "(AI|ChatGPT|모델|도구|시스템)[에를]?\s*\S*\s*(시키[다고기]|시켜|시킨다|시킬)" {output}
```

3카테고리 합산 1회 이상 → 양성. BLACK 재작성.

## 검출하지 않는 것

- 일상 자연 용례 ("그릇 깨진다·시간 죽인다·심부름 시킨다")
- 케이스가 명시적으로 영어 직역체 톤을 허용하는 경우 (해당 없음 — 한국어 케이스에서 직역체 허용은 없다)
- 사용자 입력 원문 인용에서 직역체가 사용된 경우 (BLACK이 따옴표로 인용 표시 시 무시)

## 양성 예시 5

   → 자연: "여러분의 일이 어디서부터 더 이상 통하지 않는지에 대한 이야기입니다"

3. **"가설을 더 많이 죽인다"** (c42 슬라이드 4 right h3)
   → 자연: "가설을 더 많이 흔든다" / "약한 가설을 더 많이 걸러낸다"
4. **"목요일은 자기 가설을 죽이는 날입니다"** (c42 스크립트)

5. **"자기 가설을 4명에게 반박시키기"** (c42 슬라이드 8 right list)
   → 자연: "자기 가설을 네 명에게 던져 반박받기" / "네 명이 반박하게 두기"

## 음성 예시 5

1. "그릇이 깨졌다" — 구체 사물 자연 용례 ✓
2. "시간이 비면 시간을 죽이지 말고 평가 50개를 적어 보십시오" — 일상 비유 자연 용례 ✓
3. "신입에게 슬라이드 정리를 시키지 말고 본인이 직접 30분만 다듬으십시오" — 사람-사람 사역 자연어 ✓
4. "약속이 깨질 위험이 있어서 미리 알려 드립니다" — 관계 자연 용례 ✓
5. "이번 사이클을 흔들고, 살아남는 가설만 다음 회의에 들고 가십시오" — break/kill 직역체를 자연 한국어로 대체한 정합 ✓

## 자연 표현 대체 사전

### Break 직역체 → 한국어

| 영어 직역체 | 자연 한국어 |
|---|---|
| 직무가 깨진다 | 일하는 방식이 더는 통하지 않는다 / 일이 멈춘다 |
| 모델이 깨진다 | 모델이 더는 돌아가지 않는다 / 모델이 한계에 부딪힌다 |
| 시스템이 깨진다 | 시스템이 멈춘다 / 시스템이 흔들린다 |
| 비즈니스 모델이 깨진다 | 비즈니스 모델이 더는 통하지 않는다 / 무너진다 |
| 구조가 깨지는 지점 | 구조가 흔들리는 지점 / 더는 작동하지 않는 지점 |
| 신뢰가 깨진다 | 신뢰가 무너진다 (조직·관계 자연어) |

### Kill 직역체 → 한국어

| 영어 직역체 | 자연 한국어 |
|---|---|
| 가설을 죽이다 | 가설을 흔들다 / 의심하다 / 시험하다 / 골라내다 / 접다 |
| 아이디어를 죽이다 | 아이디어를 거르다 / 내려놓다 / 접다 |
| 선택지를 죽이다 | 선택지를 좁히다 / 줄이다 |
| 옵션을 죽이세요 | 옵션을 빼세요 / 줄이세요 |
| 프로젝트를 죽이다 | 프로젝트를 접다 / 중단하다 |

### Force/Make 직역체 → 한국어

| 영어 직역체 | 자연 한국어 |
|---|---|
| AI에게 반박시키다 | AI로 반박해 보다 / AI에게 던져 반박을 받다 / AI로 흔들어 보다 |
| AI에게 분석을 시키다 | AI가 분석하게 두다 / AI에게 분석을 부탁하다 / AI를 써서 분석하다 |
| AI에게 초안 작성시키다 | AI가 초안을 쓰게 두다 / AI로 초안을 작성하다 |
| 발표하게 만들다 | 발표를 부탁하다 / 발표하도록 권하다 |

## 메타 원칙 (BLACK이 작성 시 명심)

영어 능동 강한 동사(break / kill / force / make / push / drive / leverage / unlock / kick off / nail down)가 떠올랐을 때 **한국어로 그대로 옮기지 말 것.** 한국어 비즈니스 화자는 같은 의미를 더 부드럽고 비유적인 동사 풀로 표현한다.

- 어휘 풀: 흔들다 · 접다 · 기울이다 · 다듬다 · 좁히다 · 풀다 · 붙잡다 · 받다 · 던지다 · 두다 · 멈추다 · 무너지다 · 통하지 않다 · 돌아가지 않다
- 사고: "영어로 'kill the hypothesis'를 떠올렸다 → 한국어로 '죽이다' 직역 위험"이라고 본인 입에서 깨달으면 즉시 흔들다·접다·시험하다로 대체

이 메타 원칙은 v0.4.12 "Strong-Verb Translation Trap"(영어 imperative → 한국어 강한 단음절 동사)과 함께 작동하는 두 번째 직역체 차단 축이다. v0.4.12는 *jargon 어휘* 차단, v0.4.13은 *직역 의미 연결* 차단.

## 적용 제외

- 카테고리 `소설 쓰기`(c63)·`시 쓰기`(c64) — 문학적 장치로 의도적 직역체 허용.
- 사용자 입력 원문이 영어 직역체 표현 그대로 인용된 경우(따옴표 안).
- 영어 비즈니스 용어 정의 인용 ("'kill the hypothesis'라는 영어 표현은…" 같은 메타 설명).
